笔记—HTML表单form

HTML 表单用于搜集不同类型的用户输入；将输入信息发送到服务器，并等待服务器的响应

<form></form>是用来实现前后端交互的一个重要标签，表单定义了采集数据的范围，其包含的数据内容将完整地提交给服务器

为块级标签

form常用的属性：

name：表单名称

action： 表单数据提交（submit）的地方(通常是一个后台文件名(.jsp/.php/.asp/.aspx/.py等)，或是网址)。如果是#，表提交到当前文件下。

method： 规定前端提交数据到后端的方法，常用的有get（默认）和 post

get和post的主要区别：

  • get是以字节为单位提交，只接受ASCII，而post是以字符为单位提交；

  • get以明文方式提交，把提交数据（名称/值对）加在action的URL后面并且把新的URL送至服务器，post以暗文方式提交；

  • get在浏览器中回退是无害的，而post会再次提交请求;

  • get会被浏览器主动缓存，而post不会，除非手动设置

  • get和post在传输字节数上一般没有限制，个别浏览器会有；get一般不超过2K，适合提交少量数据，post一般不超过2M

HTML新增属性：

autocomplete:on|off ：自动完成，用来帮助用户输入，每一次输入的内容，浏览器是否保存输入的值；

​ 默认为on；为了保护敏感数据（如用户账号，密码）避免本地浏览器对它们不安全存储，一般需要关闭；

​ 适用于form或部分input类型

novalidate ：如果使用该属性，则提交表单时不进行验证；适用于form或部分input类型

表单元素

<input>

单标记；根据不同的type属性，可以变化为多种状态输入方式

基本语法：<input name="" type="">

type 属性的取值及说明：

  • text：单行文本输入框（默认）

    属性：placeholder（提示）， minlength（字符的最小长度），maxlength（字符最大长度）， value（默认值），disable（失效），readonly（只读，可被激活），patten（正则匹配）

    实例：

    ​

  • password： 密码输入框 属性和text一样

  • submit： 提交按钮 将表单数据提交到后台（action位置）

  • button： 普通按钮 通常用来调用脚本文件；

    常用属性：value（按钮标题），disable（失效），onclick（绑定事件代码）

    e.g：

    <input type="button" name="button" value="注册" onclick="javascript:alert('注册');" />

  • image： 图片按钮 有提交功能，与submit功能一样；用法与button一样 有一个特殊属性：src（用来加载提示图片，替代了value）

  • radio： 单选按钮 常用属性：name（必须），value ， checked（选中），disable（失效），readonly

    使用相同的name属性值相互排斥

  • checkbox： 复选框 与radio一样

  • file：文件选择框 页面会自动添加一个文本输入框和一个“浏览…”按钮。

  • reset： 重置按钮 将表单所有组件还原初始状态

  • hidden： 隐藏按钮 在网页不可见，提交表单时，隐藏框的信息页会一起提交。属性：name，value

HTML5新增的input类型 ：

  • color： 颜色 从取色器拾取颜色

  • email：邮箱 在提交表单时，会自动验证email域的值是否合法有效

  • tel： 电话号码 （没有验证提示，主要目的有利于后期维护（语义化））

  • url： 网址输入域 提交表单时会自动验证url域的值

  • number： 数值输入域

    属性： max，maxlength，min，minlength，pattern，readonly，required，size (可见字符数)， step (合法数字间隔) ，value (规定默认值)

  • range：范围 显示滑动条

  • search：用于搜索域 例如站点搜索或谷歌搜索

Date Pickers（日期选择器）

  • date：选取日，月，年
  • month：选取月，年
  • week：选取周，年
  • time：选取时间（小时，分钟）
  • datetime： 选取时间，日，月，年（UTC时间）
  • datetime-local ：选取本地datetime

input标记的新属性：

  • height，width：只适合用于image类型的input标记的图像的高度和宽度

  • list： 规定输入域的datalist；获取焦点时，单击该域会弹出下拉列表选项。适用于类型为text，search，url，tel，email，password，date pickers，range，color的input标记。

  • placeholder：提示，描述输入域所期待的值

  • autofocus：自动获得焦点 ，设置方法为autofocus=”autofocus”或直接使用该属性autofocus。

  • required：必须输入，设置方法为required=”required”或直接使用该属性required。

  • min ，max，**step **：规定输入域所允许的最小值，最大值，合法的数字间隔，适用类型：date pickers，number，range 的input类型。

  • multiple：规定输入域可以选择多个值，适用于email，file的input标记。

  • pattern：验证模式， 属性值：正则表达式 。适用于text，search，url，tel，email，password的input标记。

  • form： 规定输入域所属的一个或多个表单，必须引用所属表单的id

    form overrides表单重写属性：

  • formmethod：修改数据提交的方法

  • formaction：修改action数据提交的地方（不常用）

  • formenctype：修改表单请求的类型（不常用）

<textarea>

文本域/多行文本框，主要用来文本的大批量输入；双标记

常用属性：

属性 值 说明
name name 定义textarea标记的名称
rows number 规定文本区内的可见行数。
cols number 规定文本区内的可见宽度。
wrap soft|hard 规定当在表单中提交时，文本区域中文本如何换行。soft表示提交时不换行（默认值）；hard表示提交时换行（包含换行符），当使用 “hard” 时，必须规定 cols 属性。
placeholder text 规定描述文本区域预期值的简短提示
minlength number 规定文本区域的最小字符数。
maxlength number 规定文本区域的最大字符数。
required required 规定文本区域是必填的。

可以通过 cols 和 rows 属性来规定 textarea 的尺寸，不过更好的办法是使用 CSS 的 height 和 width 属性。

<select>

下拉列表框，默认为单项选择，行级标签；

用**<option>**呈现每个选项；

可使用**<lable>**进行包裹做提示，也可用于其他标签

<select> 常用属性：

属性 值 说明
name name 规定下拉列表的名称。
size number 规定下拉列表中可见选项的数目
multiple multiple 可选择多项；这时变成列表框

<option>常用属性：

属性 值 说明
value text 定义送往服务器的列表选项值。
selected selected 规定选项（在首次显示在列表中时）表现为选中状态。

<button>

普通按钮，具有提交功能，

不写在form中，主要用来调用脚本文件

写入form中与submit功能一样

<lable>

为input元素定义标注；

为鼠标用户改进了可用性，当用户选择该标签时，浏览器就会自动将焦点转到和标签相关的表单控件上

属性：

属性 值 说明
for id 规定 label 绑定到哪个表单元素；需与相关元素的 id 属性相同
form form_id 规定 label 字段所属的一个或多个表单。

HTML5新增的表单元素：

<output>

定义不同类型的输出，比如脚本的输出。

属性：

属性 值 说明
for element_id(元素id) 定义输出域相关的一个或多个元素;多个元素用空格分隔
form form_id 定义输入字段所属的一个或多个表单。
name name 定义对象的唯一名称。（表单提交时使用）

浏览器兼容性：IE8 以及更早的版本不支持

实例：

<keygen>

规定用于表单的密钥对生成器字段

当提交表单时，私钥存储在本地，公钥发送到服务器，公钥可用于之后验证用户的客户端证书（client certificate）

实例：

<datalist>

定义input标记可能的选项列表，一般与input标记配合使用

datalist 及其选项不会被显示出来，它仅仅是合法的输入值列表

用户能看见一个下拉列表，里面的选项是预先定义好的，将作为用户的输入数据

input标记的list属性用来绑定datalist（list属性值应该与datalist标记的id属性值相同
